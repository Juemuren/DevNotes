# 应用开发

这里所说的应用包括 *Web/移动/桌面/命令行* 应用，嵌入式应用、游戏应用等不包含在本文里，虽然大部分知识是相通的

## 应用架构

应用不同的需求决定了其可能具有不同的架构。

最简单的架构就是没有架构，此时所有功能都由运行在用户机器上的应用实现。

常见的应用基本都是 *C/S* 架构，即 *客户端/服务端* 架构。用户只在自己的机器上运行客户端应用，它会和服务器交互以实现额外的功能。服务器会提供 *API*，即应用程序编程接口，这通常是对某种 http 请求的处理。客户端通过向服务器发送请求并接收响应，从而实现了和服务器的通信。

## 分类

应用通常会按客户端的类型进行分类

- [Web](Web.md) 应用运行在浏览器中
- **移动/桌面** 应用有图形界面，运行在移动操作系统/桌面操作系统中
- **命令行** 应用没有图形界面，运行在终端里

根据应用的需求，你可能还需要 *API*。有时你会使用现成的、由别人提供的 API；有时你可能需要自己实现一个 API。后者被称作后端开发，不过我更喜欢称之为 API 开发
